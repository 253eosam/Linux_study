1.리눅스 부팅과정
	- BIOS 실행 : ROM에 있는 바이오스 프로그램을 자동 실행 시킨다.
	- POST : 자체진단기능
	- 부트로더 실행
	- GRUB 실행 : BIOS모드는 종료되고 , GRUB가 실행되고 부팅할 OS를 선택, 커널을 찾아 메모리에 로드한 다음 실행
	- init 프로세스 실행 : init 프로세스는 /etc/inittab 파일에 설정된 내용을 실행.
2. 가상 콘솔
	- 기본적으로 6개 제공. (CentOS는 5개 사용가능 : 1번과 7번이 GUI mode(X-window) , 2~6번 CLI mode)
	/* 초기화면 init 설정 -> /etc/inittab 설정으로 콘솔 모드 변경 가능 */
3. root 계정과 일반계정
	- root : 시스템 설치 시 자동으로 생성, 시스템 관리를 위한 모든 권한을 소유
	- 일반 : 시스템관리자에  의해 생성, 시스템 관리자가 초기 패스워드를 설정 (설정하지 않으면 로그인 x ) 
		: 개인 사용 환경 변경 및 저장 가능.
4. 부팅과 종료
	- 명령어의 기본 구조 : 명령어 = 명령어 + 옵션 + 인자	// 띄어쓰기 구분 확실 , 대소문자 구분 ,
	
	/* 관리자만 사용가능 */
	- 명령어 : shoutdown [옵션] [시간] [메시지]	
		[옵션] 
		-k	: 실제로 shutdown을 하는게 아니라 모든 사용자에게 종료 메시지만 전송
		-h	: 시스템을 종료
		-r	: 시스템 재부팅
		-c	: 이미 예약되어 있는 shutdown을 취소
		ex) shutdown -h now	// 지금 종료
		ex) shutdown -r now	// 지금 재부팅
		[인수 값]
		now	: 현재 시간으로 종료
		+m	: 현재 시간으로부터 종료시점 시간 (분 단위)
		hh:mm	: 시간으로 종료시점 지정 (시간:분)
	- 명령어 : halt [옵션]
		[옵션]
		-n	: sync를 하지 않고 종료
		-d	: wtmp 파일에 기록을 남기지 않는다. (로그파일)
		-f	: 강제로 종료
	- 명령어 : init	: 리눅스 시스템이 가동되는 방법에는 0~6까지 7가지가 있다. 이를 run-level이라고 한다.
		[런레벨]
		0	: 종료 모드
		1	: 단일 사용자 모드 (시스템 복구시에 사용)
		2	: 다중 사용자 텍스트 모드 (NFS 사용 불가능)
		3	: 다중 사용자 텍스트 모드 
		4	: 사용안함
		5	: 다중 사용자 그래픽 모드 (X-Windows)
		6	: 재부팅 모드
		ex) #> init 0		// 종료
	- 명령어 : reboot : 시스템을 재부팅할 때 사용
	- 로그아웃
		- logout
		- exit
		- 단축키 Ctrl + D
5. 중요 디렉토리
	- bin	: 일반 사용자의 명령어
	- etc	: 계정 정보 , password
	- sbin	: 관리자 명령어		// 관리자만 사용 가능한 명령어
	- var	: 웹서버, 로그
	- home	: 각 계정의 활동 공간
	- root	: root의 홈 디렉토리
	- tmp 	: 임시 디렉토리

1. 디렉토리 구조

/	
	- 루트 디렉토리 (최상위 디렉토리)
	- 이 디렉토리를 기준으로 모든 디렉토리가 생성된다.
	- 모든 디렉토리의 출발점

/bin
	- 일반 사용자들을 위한 명령어들이 있다. (기본 명령어)

/boot
	- 부트로더와 부팅을 위한 파일들이 있다.

/dev
	- 장치파일들이 있다.(devices)
	- 하드 디스크, ROM 등 각종 장치디바이스 등을 포함

/etc
	- 시스템 또는 프로그램의 환경설정 파일들이 있다.
	- 피일시스템, User, Group, init, port 정보, 로그인 메시지, 언어, 방화벽, 네트워크 관리 등의 정보

/home
	- 일반 사용자의 홈 디렉토리가 생성되는 곳

/lib
	- 시스템 운영 및 프로그램 구동할 때 필요한 공유 라이브러리

/root
	- 시스템 관리자, 즉 root 사용자의 홈 디렉토리

/mnt
	- CD-ROM과 플로피 장치에 대한 기본 마운트 포인트를 제공

/sbin
	- 관리자가 사용하는 시스템 운영에 필요한 명령어가 존재하는 디렉토리 (fdisk, reboot 등)

/tmp
	- 프로세스의 생성 과정에서 발생되는 임시 파일을 저장하는 디렉토리

/user
	- 프로그램을 설치할 때 패키지에 포함된 대부분의 파일이 이곳에 설치
	- 윈도우 Programs files 폴더와 유사

/var
	- 내용이 자주 변경되는 가변 자료가 저장
		- 시스템 운영할 때 발생되는 로그(log)
		- 메일 송/수신 할 때 임시로 저장되는 스풀(spool)
		- 웹 서버 관련 자ㄹ (/var/www)
		- 데이터 베이스 (MySQL)

<절대경로 , 상대경로> (.. 부모) , (. 현재)
- 절대 경로 : 최상위 디렉토리를 기준으로 이동경로 나열 ( 최상위 디렉토리)
- 상대 경로 : 현재 위치에서 이동
/* 
경로 설정시 상대경로와 절대경로를 적절하게 사용하여 경로가 헷갈리지 않겠끔 한다. 
현재 경로에 대한 생략을 하지말고 './'를 사용하자!
*/
2. 기본 명령어 
	1) pwd 	: 현재 경로 보기 

	2) cd	: 디렉토리 이동 
		#> cd [인수 값]

		[인수 값]
		Directory : 이동하기 원하는 디렉토리 
		.	: 현재 디렉토리 
		..	: 상위 디렉토리 
		~	: 로그인 사용자의 홈디렉토리 
		-	: 이전 작업 디렉토리 

	문제1) tmp 디렉토리로 이동 (절대경로) 
	문제2) init.d 디렉토리로 이동 (상대경로)

	3) ls 	: 파일 내역 출력 
		#> ls [옵션] [디렉토리/파일]
	
		[옵션]
		-a	: 모든파일/디렉토리 표시 
		-l	: 자세히 출력 
		-d	: 디렉토리 정보 출력 
		-n	: UID, GID 출력 
		-R	: 하위 경로와 모든 파일 나열 

	4) cp 	: 파일, 디렉토리 복사 
		#> cp [옵션] [복사할파일] [복사할위치]

		[옵션]
		-i	: 복사에 대한 실행 여부 물음 
		-f	: 강제 복사
		-r	: 하위 디렉토리와 파일 모두 복사 
		-v	: 복사 진행 상태 출력 
		-a	: 원본 파일의 속성, 링크 정보 유지 복사 

	5) mv	: 파일, 디렉토리 이동 
		#> mv [옵션] [이동할파일] [이동할위치]

		[옵션]
		-i	: 이동에 대한 실행여부 물음
		-f	: 강제 이동 
		-v	: 이동 진행 상태 출력 

	6) mkdir 	: 디렉토리 생성 
		#> mkdir [옵션] [디렉토리명]

		[옵션]
		-p	: 상위 디렉토리 생성 

	7) rmdir	: 디렉토리 삭제 
		#> rmdir [옵션] [디렉토리명]

		[옵션]
		-p 	: 상위 디렉토리 삭제 

	8) rm	: 파일, 디렉토리 삭제 
		#> rm [옵션] [디렉토리/파일]

		[옵션]
		-f	: 강제 삭제
		-r	: 디렉토리 삭제시 하위 경로와 파일 삭제 
		-v	: 파일 삭제 정보 자세하게 출력 
		-i	: 삭제에 대한 실행여부 물음 
	** 주의사항 
		> rm -rf /

	9) cat	: 파일의 내용을 화면에 출력 
		#> cat [파일 이름]

		[옵션]
		-n	: 줄번호를 화면 왼쪽에 나타낸다. 공백을 포함한다. 
		> 	: 새로 만들거나 덮어씌운다. 
		>> 	: 이어쓰기 

	10) touch : 파일 생성 및 시간 정보 변경 
		#> touch [옵션] [파일명]

		[옵션]
		-t	: 지정 시간으로 변경 <MMDDhhmm>
		-r	: 시간동기화

	11) head	: 파일 내용 중 처음부터 10줄 출력 
		#> head [파일명]

	12) tail	: 파일 내용 중 마지막부터 10줄 출력 
		#> tail [파일명]

		[옵션]
		-f	: 실시간으로 작성되는 내용 보기 

	13) man 	: 명령어의 사용법 
		#> man 명령어 

	14) more : 파일 내용을 화면단위로 출력 
		#> more [파일명]

	15) less 	: more를 좀 더 보완한 명령어 
		#> less [파일명]

	16) file	: 파일 종류 확인 
		#> file [파일명]
		ex) 어벤져스.avi 
		
		/root 		: 디렉토리 
		/etc/passwd 	: ASCII text
		/bin/ls 		: executable
		/dev/sda		: 장치파일 

	17) history : 입력했던 명령어 확인 
		#> history [옵션] [숫자]

		[옵션]
		-d	: 입력했던 명령어 개별 삭제 
		-c	: 입력했던 명령어 모두 삭제 

	리눅스의 특별기능)
