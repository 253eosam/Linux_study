[링크파일]
	- 하드 링크 파일
	- 소프트 링크 파일 (심볼릭 링크 파일)
	
1. I-Node
	> 파일 경로 정보를 제외한 모든 정보를 가지고 있다.
	> 실제데이터와 파일이름을 연결시켜주는 다리 역할


2. 하드 링크 파일
	1) 동일한 inode를 사용
	2) 원본 수정 시 하드링크도 수정된다. 
	3) 원본 삭제 시 살아 있음.
	4) 원본의 이동이나 삭제 시 남아 있다. 
	5) 파일 복사와 차이는 공간을 차지하지 않는다.
	
	
3. 심볼릭 링크 파일
	1) window 바로가기 아이콘과 비슷
	2) 주로 사용자가 많이 사용
	3) 긴 디렉토리명 이동시 유용 ex) /tmp/abc/hello/bbb/ddd/test/hi
	4) 원본을 다른 곳으로 이동시키면 사용불가	
	5) 새로운 indoe 생성
	6) 원본 자체 포인터 역할뿐이므로, 원본 수정 시 수정된 파일이 보인다
	7) 원본의 삭제 시 링크가 끊긴다.

4. 링크 파일 명령어
	#> ln <src_path> <link_path>
	#> ln -s <src_path> <link_path>

문제2) 		1. 하드링크와 심볼릭링크의 허가권을 변경 하였을 때 원본파일의 변화는?
			-> 하드링크의 허가권은 원본 파일과 같이 변경된다.
			-> 심볼릭링크의 허가권은 원본 파일의 권한과는 별개로 777 권한을 고수한다.

		2. 디렉토리에 하드링크가 걸리는가? 안되면 안되는 이유는 무엇인가?
			-> 디렉토리에는 하드링크가 걸리지 않는다.
			-> 두개의 디렉토리 중 하나의 i-node를 가리키게 되면 두 디렉토리 중 하나가 경로를 변경시 무한 루프가 발생 할 수 있기 때문이다.

		3. 디렉토리에 심볼릭링크가 걸리는가? 되는 이유?
			-> 심볼릭링크는 디렉토리의 경로 위치만을 가지고 있기 때문에 가능하다. 또한 무한루프시 몇개 이상의 symlink를 거치게 되면 something wrong으로 판단하고 멈추게 된다.

		4. 하드링크에 심볼릭 링크가 걸리는가?
			-> 가능하다. 하드링크의 심볼릭링크의 허가권을 변경시 원본 파일의 허가권도 함께 변경된다. 

		
		5. 심볼릭 링크 파일에 하드링크가 걸리는가?
			-> 가능하다. 심볼릭 링크의 원본 파일에 대한 하드링크가 걸린다.
